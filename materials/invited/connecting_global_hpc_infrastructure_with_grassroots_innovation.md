## 🎯 この発表を理解するための前提知識

### 1️⃣ HPCとは何か？なぜ教育が重要なのか？

**HPC（High Performance Computing）** とは、複雑な問題を高速に解決するための計算技術です。

```
┌────────────────────────────────────────────────────────────────┐
│                    HPC が活躍する分野                           │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐         │
│  │  気象予報    │  │  創薬開発    │  │  自動車設計  │         │
│  │  台風の進路  │  │  新薬候補の  │  │  衝突シミュ  │         │
│  │  予測        │  │  探索        │  │  レーション  │         │
│  └──────────────┘  └──────────────┘  └──────────────┘         │
│                                                                │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐         │
│  │  宇宙物理    │  │  AI/機械学習 │  │  金融リスク  │         │
│  │  銀河形成の  │  │  大規模言語  │  │  市場シミュ  │         │
│  │  シミュレ    │  │  モデル学習  │  │  レーション  │         │
│  └──────────────┘  └──────────────┘  └──────────────┘         │
│                                                                │
│                ↓ これらすべてに共通する課題 ↓                  │
│                                                                │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │     「HPCを使いこなせる人材が世界的に不足している」       │ │
│  └──────────────────────────────────────────────────────────┘ │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

**高校生向け解説**：
スーパーコンピュータは「すごい計算機」ですが、それを使いこなすには特別な知識が必要です。普通のプログラミングとは違い、何千台ものコンピュータを同時に動かす技術が求められます。しかし、この技術を教えられる人も、学べる場所も限られています。

かつてHPCは一部のエリート研究機関だけのものでしたが、クラウドや共有インフラにより、より多くの人がアクセスできるようになっています。

```
│                    HPC 民主化の歴史                             │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  【過去】エリート研究機関限定                                   │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │  ● 国立研究所（米国の国立研究所、日本の理研など）         │ │
│  │  ● 大手企業の研究開発部門                                 │ │
│  │  ● トップ大学の一部研究室                                 │ │
│  └──────────────────────────────────────────────────────────┘ │
│                          ↓                                     │
│  【現在】アクセスの拡大                                         │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │  ● クラウドHPC（AWS, Google Cloud, Azure）                │ │
│  │  ● 地域スパコンセンター（NCI, NSCC Singapore等）           │ │
│  │  ● 大学の共同利用システム                                 │ │
│  │  ● 教育用クラスタ（Raspberry Pi クラスタなど）            │ │
│  └──────────────────────────────────────────────────────────┘ │
│                          ↓                                     │
│  【課題】アクセスだけでは不十分                                 │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │  ❌ マシンがあっても使い方がわからない                    │ │
│  │  ❌ 教えられる人材が不足                                  │ │
│  │  ❌ 体系的なカリキュラムがない                            │ │
│  │  ❌ 理論と実践のギャップ                                  │ │
│  └──────────────────────────────────────────────────────────┘ │
│                          ↓                                     │
│  【解決策】Grassroots Innovation（草の根イノベーション）        │
│  ┌──────────────────────────────────────────────────────────┐ │
│  │  ✓ 学生コンペティション（APAC HPC-AI Competition等）      │ │
│  │  ✓ 国際サマースクール（IHPCSS, ACM Summer School）        │ │
│  │  ✓ メンター制度とコミュニティ                             │ │
│  │  ✓ オープン教材とハンズオン研修                           │ │
│  └──────────────────────────────────────────────────────────┘ │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

**高校生向け解説**：
「民主化」とは、特別な人だけでなく、誰でもアクセスできるようにすること。インターネットが情報の民主化を実現したように、HPCも教育を通じてより多くの人が使えるようになりつつあります。

### 3️⃣ 並列プログラミングの基礎

HPCを使うには、**並列プログラミング**という特殊な技術が必要です。

```
┌────────────────────────────────────────────────────────────────┐
│                逐次処理 vs 並列処理                             │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│  【逐次処理】1つずつ順番に処理                                  │
│  ┌────┐ → ┌────┐ → ┌────┐ → ┌────┐                          │
│  │ A  │   │ B  │   │ C  │   │ D  │                          │
│  └────┘   └────┘   └────┘   └────┘                          │
│           合計時間: 4単位                                      │
│                                                                │
│  【並列処理】同時に複数を処理                                   │
│  ┌────┐ ┌────┐ ┌────┐ ┌────┐                                │
│  │ A  │ │ B  │ │ C  │ │ D  │                                │
│  └────┘ └────┘ └────┘ └────┘                                │
│           合計時間: 1単位（理想的には）                        │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

**主要な並列プログラミング技術**：

| 技術 | 用途 | 難易度 |
|------|------|--------|
| **OpenMP** | 1台のコンピュータ内で複数CPUコアを使う | ⭐⭐ |
| **MPI** | 複数台のコンピュータを協調させる | ⭐⭐⭐⭐ |
| **CUDA/OpenCL** | GPUで大量の計算を並列実行 | ⭐⭐⭐ |

**簡単なOpenMPの例**（C言語）：
```c
// 通常のfor文
for (int i = 0; i < 1000000; i++) {
    result[i] = compute(data[i]);
}

// OpenMPで並列化（たった1行追加！）
#pragma omp parallel for
for (int i = 0; i < 1000000; i++) {
    result[i] = compute(data[i]);
}
```

**高校生向け解説**：
普通のプログラムは「1人で仕事をする」イメージ。並列プログラムは「チームで分担して仕事をする」イメージです。ただし、チームワークには「誰が何をやるか決める」「結果を集める」「順番を調整する」といった追加の作業が必要になります。

## 🔬 技術的詳細

### 🧪 現在のHPC教育の課題

| 課題 | 詳細 | 影響 |
|------|------|------|
| **スキルギャップ** | 65%の組織がHPC人材不足を報告 | 研究・開発の遅延 |
| **教育リソースの偏在** | 先進国に集中、新興国では限定的 | グローバルな不平等 |
| **カリキュラムの標準化不足** | 各機関がバラバラの内容を教える | 人材の質のばらつき |
| **実機経験の不足** | シミュレータでは学べないことが多い | 実務対応力の欠如 |
| **AI/HPC融合の遅れ** | 両分野を統合できる人材が希少 | 次世代技術開発の障壁 |

### 📈 主要な教育イニシアティブ

**EuroHPC EVITA (2025年4月開始)**：
- 予算: 600万ユーロ（Digital Europe Programme）
- 期間: 4年間
- 目標: 欧州全域でHPC教育を標準化
- 参加: 7カ国の機関（BSCがコーディネーター）
- 内容: 能力フレームワーク、モジュール式教材開発

**APAC HPC-AI Competition (第8回/2025年)**：
- 参加: アジア太平洋地域から49チーム
- 課題: NWChem最適化 + DeepSeek-R1推論
- 優勝: 台湾・清華大学（4年連続）
- 特徴: 6ヶ月間の実践的トレーニング

**International HPC Summer School**：
- 2025年: リスボン（ポルトガル）
- 2026年: パース（オーストラリア）
- 対象: 日本、欧州、米国、豪州、カナダの学生
- 費用: 全額支給

### 🔍 各地域の特色

| 地域 | 特徴的な取り組み | 強み |
|------|------------------|------|
| **欧州** | EuroHPC JU、EVITA、EUMaster4HPC | 統一フレームワーク |

| **アジア太平洋** | APAC Competition、PRAGMA | 多様性と急成長 |

## 📖 用語集（高校生向け解説付き）

| 用語 | 読み方 | 説明 |
|------|--------|------|
| **HPC** | エイチピーシー | High Performance Computing。スーパーコンピュータを使った高性能計算。普通のパソコン1台で100年かかる計算を、数時間で終わらせる技術。 |
| **スーパーコンピュータ** | - | 何千〜何万台のコンピュータを束ねて超高速計算を行う装置。日本の「富岳」は世界トップクラス。 |
| **並列計算** | へいれつけいさん | 複数のコンピュータやCPUを同時に動かして計算を分担する方法。「1人で10日かかる仕事を10人で1日で終わらせる」イメージ。 |
| **MPI** | エムピーアイ | Message Passing Interface。複数のコンピュータ間でデータをやり取りするための「共通言語」。LINEのグループトークのように情報を共有する。 |
| **OpenMP** | オープンエムピー | 1台のコンピュータ内で複数のCPUコアを使うための技術。プログラムに「ここは分担して」と印をつけるだけで並列化できる。 |
| **CUDA** | クーダ | NVIDIAのGPUでプログラムを動かすための技術。元はゲーム用だったGPUを計算に使う方法を提供。 |
| **クラスタ** | - | 複数のコンピュータをネットワークで接続し、1つの大きなコンピュータのように使う仕組み。「コンピュータの集合体」。 |
| **ノード** | - | クラスタを構成する個々のコンピュータ。スパコンは何千もの「ノード」で構成される。 |
| **サマースクール** | - | 夏に開催される集中講座。1週間程度で特定分野を深く学ぶ。世界中の学生が参加し、交流する機会にもなる。 |
| **メンター** | - | 経験豊富な先輩が後輩を指導・支援する人。技術だけでなく、キャリアの相談にも乗ってくれる存在。 |
| **コンペティション** | - | 競技会。学生同士がチームを組んでHPC課題に挑戦し、性能を競う。実践的なスキルが身につく。 |
| **APAC** | エイパック | Asia-Pacific。アジア太平洋地域。日本、中国、韓国、東南アジア、オーストラリアなどを含む。 |
| **BSC** | ビーエスシー | Barcelona Supercomputing Center。スペインの国立スパコンセンター。MareNostrumスパコンで有名。 |
| **JSC** | ジェイエスシー | Jülich Supercomputing Centre。ドイツの主要スパコンセンター。欧州最大級の施設。 |
| **R-CCS** | アールシーシーエス | RIKEN Center for Computational Science。理化学研究所の計算科学研究センター。「富岳」を運用。 |
| **EuroHPC JU** | ユーロエイチピーシージェイユー | European High Performance Computing Joint Undertaking。欧州のHPC推進機関。 |
| **EVITA** | エビタ | EuroHPC Virtual Training Academy。欧州全体のHPC教育を標準化するプロジェクト（2025年開始）。 |
| **IHPCSS** | アイエイチピーシーエスエス | International HPC Summer School。日米欧豪加の大学院生向け国際サマースクール。 |
| **スキルギャップ** | - | 企業が必要とするスキルと、人材が持つスキルの差。HPC分野では深刻な問題。 |
| **民主化** | みんしゅか | 特定の人だけでなく、誰でもアクセス・利用できるようにすること。HPCの民主化=より多くの人がスパコンを使えるようになること。 |
| **草の根イノベーション** | くさのねイノベーション | 大企業や政府主導ではなく、現場の人々が自発的に生み出す革新。ボトムアップの変革。 |
| **Grassroots** | グラスルーツ | 草の根。一般市民や現場レベルから始まる活動。「トップダウン」の反対。 |
| **パイプライン** | - | 人材や製品が流れていく仕組み。「教育パイプライン」=学生が学び、育ち、就職するまでの流れ。 |
| **ハンズオン** | - | 実際に手を動かして学ぶこと。座学（講義を聞くだけ）の反対。実機を使った実習。 |
| **カリキュラム** | - | 教育課程。何を、どの順番で、どう教えるかを決めた計画。 |
| **産学連携** | さんがくれんけい | 企業（産）と大学（学）が協力すること。研究や人材育成で協力し、双方にメリットがある。 |
