## 🎯 この発表を理解するための前提知識

### 1️⃣ フィッシュボウルパネルとは

**フィッシュボウル（金魚鉢）パネル**は、従来のパネルディスカッションを革新した参加型の議論形式です。

```
┌────────────────────────────────────────────────────────────┐
│                フィッシュボウル形式の構造                   │
├────────────────────────────────────────────────────────────┤
│                                                            │
│              ┌─────────────────────┐                       │
│              │    中央の「金魚鉢」   │                       │
│              │  ┌───┐ ┌───┐ ┌───┐ │                       │
│              │  │ P1│ │ P2│ │ P3│ │ ← パネリスト3名        │
│              │  └───┘ └───┘ └───┘ │                       │
│              │        ┌───┐       │                       │
│              │        │空席│       │ ← 聴衆が座れる空き席   │
│              │        └───┘       │                       │
│              └─────────────────────┘                       │
│                        ▲                                   │
│              ┌─────────┴─────────┐                         │
│              │                   │                         │
│        ┌───────────────────────────────┐                   │
│        │           聴衆席              │                   │
│        │   ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○    │                   │
│        │   ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○    │                   │
│        │   ○ ○ ○ ○ ○ ○ ○ ○ ○ ○ ○    │                   │
│        └───────────────────────────────┘                   │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

#### 進行ルール（SCA/HPCAsia版）

| 時間 | 内容 |
|------|------|
| **8分ごと** | パネリスト1名が退場、聴衆から1名が参加 |
| **交代方式** | モデレーターが指名またはボランティア方式 |
| **雰囲気** | 「礼節ある意見対立」を推奨 |

**日常的な比喩**: フィッシュボウルパネルは、テレビの討論番組に「一般視聴者が電話参加できる」仕組みを追加したようなものです。専門家だけでなく、会場の誰もが議論に直接参加できます。

**日常的な比喩**: HPC市場は、自動車市場に例えると分かりやすいです。ハイパースケール企業（Google、Meta等）は「大企業の社用車フリート」、研究機関は「レーシングチーム」、一般企業は「個人の車」のような位置づけです。

### 3️⃣ HPC と AI の融合（コンバージェンス）

近年、HPCとAIは急速に融合しています。

```
         従来のHPC                    従来のAI
    ┌─────────────────┐        ┌─────────────────┐
    │ 科学シミュレーション │        │ 画像認識・NLP    │
    │ 気象予測          │        │ 推薦システム      │
    │ 分子動力学        │        │ 機械学習         │
    │ 流体解析          │        │ ディープラーニング │
    └────────┬────────┘        └────────┬────────┘
             │                          │
             └──────────┬───────────────┘
                        │
                        ▼
              ┌─────────────────────┐
              │    HPC-AI 融合      │
              ├─────────────────────┤
              │ ・AI for Science    │
              │ ・科学的機械学習    │
              │ ・大規模言語モデル訓練│
              │ ・シミュレーション+ML │
              │ ・Foundation Models  │
              └─────────────────────┘
```

#### 融合の実態（2024年調査）

| 指標 | 数値 |
|------|------|
| HPCアプリにAI/MLを統合している | **75%** |
| AI/MLはHPCと別々に運用 | 30% |

**日常的な比喩**: HPCとAIの融合は、「電気自動車」の登場に似ています。従来の「ガソリン車（HPC）」と「電気モーター（AI）」が組み合わさり、「ハイブリッド」や「完全電動」のような新しい形態が生まれています。

### 4️⃣ 業界アナリストの役割

**業界アナリスト**は、技術トレンド、市場動向、競争環境を分析し、企業や投資家に洞察を提供する専門家です。

```
┌────────────────────────────────────────────────────────────┐
│             業界アナリストの主な活動                        │
├────────────────────────────────────────────────────────────┤
│                                                            │
│   ┌──────────────┐    ┌──────────────┐    ┌──────────────┐│
│   │ 市場調査     │    │ データ分析    │    │ 予測・提言   ││
│   │              │───▶│              │───▶│              ││
│   │ ・ユーザー調査│    │ ・売上データ  │    │ ・5年予測    ││
│   │ ・ベンダー取材│    │ ・技術動向    │    │ ・戦略提言   ││
│   │ ・カンファレンス│   │ ・競合分析    │    │ ・リスク分析 ││
│   └──────────────┘    └──────────────┘    └──────────────┘│
│                                                            │
└────────────────────────────────────────────────────────────┘
```

**日常的な比喩**: 業界アナリストは、スポーツの「解説者」のような存在です。選手（企業）のパフォーマンスを分析し、試合（市場）の流れを読み、今後の展開を予想します。

### 5️⃣ 「ハイプ」と「リアリティ」を見極める

技術業界では、過大な期待（**ハイプ**）と実際の価値（**リアリティ**）を区別することが重要です。

```
┌─────────────────────────────────────────────────────────────┐
│                  ガートナーのハイプサイクル                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   期待                                                      │
│    ▲              ○ 過度な期待の                           │
│    │             /  ピーク期                                │
│    │            /                                           │
│    │           / ← ここで多くの技術が                       │
│    │          /    「幻滅」される                           │
│    │    ○───○                                              │
│    │   /      ＼        ○──○──○ 生産性の                  │
│    │  /        ＼      /         安定期                     │
│    │ /          ＼    /                                     │
│    │○            ＼  /                                      │
│    │黎明期        ○ 幻滅期                                  │
│    │              （谷底）                                  │
│    └────────────────────────────────────────────▶ 時間      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### AI・HPCにおける「ハイプ vs リアリティ」の例

| トピック | ハイプ（過大評価） | リアリティ（実態） |
|---------|------------------|------------------|
| **汎用AI（AGI）** | 「数年で人間を超える」 | まだ道半ば、特定タスクでは優秀 |
| **量子コンピュータ** | 「すべての問題を解決」 | 特定の問題で優位性、実用化は進行中 |
| **クラウドHPC** | 「オンプレ不要」 | 利用率次第、~25%浸透で均衡 |
| **GPU独占** | 「NVIDIA一強永続」 | AMD、新興企業が追撃中 |

## 📖 発表概要

### 🎯 セッションの目的

このフィッシュボウルパネルは、**HPCとAIが直面する「破壊的変化」と「革命」**を探求し、真のイノベーションと単なるマーケティング・ハイプを見分けることを目指します。

多様な業界リーダーから幅広い意見を引き出すよう設計されており、聴衆も直接議論に参加できるユニークな形式です。

### 📊 期待される議論トピック

ISC23・ISC24での実績を踏まえ、以下のようなトピックが議論されると予想されます：

1. **AI・HPC融合の真実**
   - 本当に融合しているのか？それとも別々の世界か？
   - 「AI for Science」は期待通りの成果を出しているか？

2. **エクサスケール時代の課題**
   - エクサスケールは何を変えたか？
   - 次のターゲットは？

3. **GPU覇権とオルタナティブ**
   - NVIDIAの支配は続くか？
   - AMD、Intel、新興企業のチャンス

4. **クラウド vs オンプレミス**
   - どちらがコスト効率が良いか？
   - データ主権・セキュリティの観点は？

5. **国家戦略としてのHPC-AI**
   - 「HPCナショナリズム」の台頭
   - 各国のソブリンAI戦略

6. **人材・スキルギャップ**
   - 誰がこれらのシステムを運用するのか？
   - HPCの民主化は進むか？

7. **持続可能性・電力問題**
   - AIの電力消費問題
   - グリーンHPCの可能性

### 🔍 技術的課題と機会

#### 課題

| 課題 | 詳細 |
|------|------|
| **メモリウォール** | プロセッサ速度にメモリ帯域が追いつかない |
| **電力消費** | スパコンが小さな町の電力を消費 |
| **精度の問題** | AIは16/8bit、科学計算は64bit精度が必要 |
| **コスト高騰** | 最新スパコンは$10B規模に |
| **アクセシビリティ** | SSHとCLIの壁、専門知識が必要 |

#### 機会

| 機会 | 詳細 |
|------|------|
| **マルチチップレット** | 歩留まり向上、大規模ダイの実現 |
| **先端プロセス** | 3nm→2nm→1nm以下へ |
| **新興アーキテクチャ** | Cerebras、Groq、SambaNova等 |
| **HPCの民主化** | クラウドとツールの進化 |

| 用語 | 読み方 | 説明 |
|------|--------|------|
| **フィッシュボウルパネル** | ふぃっしゅぼうるぱねる | パネリストが定期的に聴衆と入れ替わる討論形式。「金魚鉢」の中で議論しているように、周囲から観察しながらいつでも参加できる。テレビの視聴者参加型討論のライブ版。 |
| **HPC** | エイチピーシー | High Performance Computing の略。スーパーコンピュータを使った高性能計算。天気予報、薬の開発、宇宙シミュレーションなど、普通のパソコンでは計算できない問題を解く。 |
| **業界アナリスト** | ぎょうかいあなりすと | 市場動向を分析し、将来を予測する専門家。スポーツ解説者が試合を分析するように、業界の「試合展開」を読み解く。 |
| **ハイパースケール** | はいぱーすけーる | Google、Meta、Amazon、Microsoftなどの巨大IT企業が運営する超大規模データセンター。一社で数万〜数十万台のサーバーを持つ。 |
| **ハイプ** | はいぷ | 技術やトレンドに対する過大な期待や誇大宣伝。「バズワード」が先行し、実態が伴わないことも多い。流行りのダイエット法が「すぐ痩せる！」と宣伝されるのに似ている。 |
| **エクサスケール** | えくさすけーる | 毎秒100京回（10の18乗回）の計算ができる性能レベル。Frontier（米国）やAurora（米国）がこのレベルに到達。 |
| **GPU** | ジーピーユー | Graphics Processing Unit。もともとゲームの画像処理用だが、AI計算でも大活躍。多数の簡単な計算を同時にできる。 |
| **オンプレミス** | おんぷれみす | 自社でサーバーやコンピュータを所有・運用すること。「自家用車」を持つようなもの。 |
| **クラウド** | くらうど | インターネット経由で借りるコンピュータ資源。「レンタカー」のように必要な時だけ使う。 |
| **ソブリンAI** | そぶりんえーあい | 国家主権を重視したAIインフラ。外国のクラウドに依存せず、自国内でAIを運用する動き。 |
| **データ主権** | でーたしゅけん | 自国のデータは自国で管理すべきという考え方。パスポートや戸籍を外国に預けないのと同じ発想。 |
| **CAGR** | シーエージーアール | Compound Annual Growth Rate。年平均成長率。毎年平均何%ずつ成長するかを示す指標。 |
| **TOP500** | トップ500 | 世界のスーパーコンピュータ性能ランキング。毎年6月と11月に発表。 |
| **ISC** | アイエスシー | International Supercomputing Conference。毎年ドイツで開催される欧州最大のHPCカンファレンス。 |

| **マルチチップレット** | まるちちっぷれっと | 複数の小さなチップを組み合わせて1つの大きなプロセッサを作る技術。レゴブロックのように組み合わせる。 |
| **プロセスノード** | ぷろせすのーど | 半導体製造技術の世代。3nm、5nm、7nmなど。数字が小さいほど微細で高性能。 |
| **ガートナーハイプサイクル** | がーとなーはいぷさいくる | 新技術が過度な期待→幻滅→安定した普及に至る過程を示すモデル。 |
