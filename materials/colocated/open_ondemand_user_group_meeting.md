## 🎯 この発表を理解するための前提知識

### 1️⃣ HPC (High Performance Computing) とは？

**簡単に言うと**: 普通のパソコンでは何年もかかる計算を、数時間〜数日で終わらせる「超高速計算機」のことです。

```
🖥️ 普通のパソコン        🏢 スーパーコンピュータ
   1台で計算               数千〜数万台が協力して計算

例: 天気予報の計算
┌─────────────┐          ┌─────────────────────────────┐
│   1台で     │          │ 🖥️🖥️🖥️🖥️🖥️🖥️🖥️🖥️ │
│   100年     │    →     │      数時間で完了           │
│   かかる    │          │  （並列処理の力）           │
└─────────────┘          └─────────────────────────────┘
```

**なぜ必要か**:
- 気象予報・気候変動シミュレーション
- 新薬の分子設計
- 自動車の衝突シミュレーション
- AI・機械学習の大規模トレーニング

### 2️⃣ HPCへのアクセス方法の変遷

**昔のやり方**: コマンドライン（黒い画面に文字を打ち込む）

```
従来のHPCアクセス方法

ユーザー                    HPCクラスタ
  │                            │
  │  ssh username@hpc.edu      │
  │ ─────────────────────────→ │
  │                            │
  │  [黒い画面]                │
  │  $ cd /work/project        │
  │  $ sbatch job_script.sh    │
  │  $ squeue -u username      │
  │                            │

問題点:
❌ Linuxコマンドを覚える必要がある
❌ ジョブスクリプトを手書きする必要がある
❌ 結果の可視化が大変
❌ ファイル転送にscp/sftpが必要
```

**現代のやり方**: Webブラウザでアクセス

```
Open OnDemandによるアクセス

ユーザー                    Open OnDemand            HPCクラスタ
  │                            │                        │
  │  https://ondemand.hpc.edu  │                        │
  │ ─────────────────────────→ │                        │
  │                            │    内部で連携          │
  │  [Webブラウザ]             │ ←────────────────────→ │
  │   📁 ファイル管理          │                        │
  │   📊 Jupyter Notebook      │                        │
  │   🖥️ 仮想デスクトップ     │                        │
  │   📝 ジョブ投入            │                        │

利点:
✅ ブラウザだけで全て完結
✅ クリック操作でジョブ投入
✅ 直接ファイルのアップロード/ダウンロード
✅ グラフィカルなアプリを利用可能
```

### 3️⃣ Open OnDemandとは？

**簡単に言うと**: スーパーコンピュータを「Webサイトのように」使えるようにするオープンソースソフトウェアです。

**身近な例で説明**:
- Google DriveやDropboxでファイル管理するように、HPCのファイルを管理
- Gmailを開くように、ブラウザでHPCにログイン
- オンラインゲームのように、グラフィカルなアプリを使用

```
Open OnDemandの主要機能

┌────────────────────────────────────────────────────────┐
│                  Open OnDemand ダッシュボード          │
├────────────────────────────────────────────────────────┤
│                                                        │
│  📁 Files          📊 Jobs          🖥️ Apps           │
│  ┌──────────┐     ┌──────────┐     ┌──────────┐       │
│  │ホーム    │     │投入済み  │     │Jupyter   │       │
│  │プロジェクト│    │実行中    │     │RStudio   │       │
│  │結果      │     │完了      │     │MATLAB    │       │
│  │アップロード│    │テンプレート│    │VSCode   │       │
│  └──────────┘     └──────────┘     └──────────┘       │
│                                                        │
│  💻 Shell Access    📈 Active Jobs    ⚙️ Settings     │
│                                                        │
└────────────────────────────────────────────────────────┘
```

### 4️⃣ Webアプリケーションの仕組み

Open OnDemandは**Webアプリケーション**として動作します。

```
Webアプリケーションとは？

┌─────────────┐         ┌─────────────┐         ┌─────────────┐
│  ブラウザ   │ ──────→ │ Webサーバー │ ──────→ │ データベース│
│  (Chrome等) │ ←────── │  (Apache等) │ ←────── │   (MySQL等) │
└─────────────┘  HTTP   └─────────────┘         └─────────────┘
   ユーザー側           サーバー側                データ保存

例えば:
- Amazon = 商品情報をWebで表示
- YouTube = 動画をWebで配信
- Open OnDemand = HPCをWebで操作
```

**Open OnDemandの技術スタック**:

```
Open OnDemand アーキテクチャ

ユーザー
   │
   ▼ HTTPS
┌──────────────────────────────────────────────┐
│  Apache HTTP Server + mod_auth (認証)        │
└──────────────────────────────────────────────┘
   │
   ▼
┌──────────────────────────────────────────────┐
│  Per-User NGINX (PUN)                        │
│  ユーザーごとに専用のプロセスが起動          │
└──────────────────────────────────────────────┘
   │
   ▼
┌──────────────────────────────────────────────┐
│  Phusion Passenger (アプリサーバー)          │
│  Ruby / Python / Node.js アプリを実行        │
└──────────────────────────────────────────────┘
   │
   ▼
┌──────────────────────────────────────────────┐
│  Ruby on Rails アプリケーション群            │
│  - Dashboard (ダッシュボード)                │
│  - File Browser (ファイル管理)               │
│  - Job Composer (ジョブ投入)                 │
│  - Interactive Apps (Jupyter等)              │
└──────────────────────────────────────────────┘
   │
   ▼
┌──────────────────────────────────────────────┐
│  HPC クラスタ                                │
│  - Slurm / PBS / LSF (ジョブスケジューラ)   │
│  - 計算ノード                                │
│  - ストレージ                                │
└──────────────────────────────────────────────┘
```

### 5️⃣ ジョブスケジューラとは？

**簡単に言うと**: スーパーコンピュータの「順番待ちシステム」です。

```
ジョブスケジューラの役割

            ユーザーA    ユーザーB    ユーザーC
               │           │           │
               ▼           ▼           ▼
          ┌─────────────────────────────────┐
          │      ジョブスケジューラ          │
          │   (Slurm / PBS / LSF など)      │
          │                                 │
          │  待ち行列 (Queue)               │
          │  ┌─────┬─────┬─────┬─────┐     │
          │  │Job1 │Job2 │Job3 │ ... │     │
          │  └─────┴─────┴─────┴─────┘     │
          │                                 │
          │  優先度・リソース要求を考慮     │
          │  して計算ノードに割り当て       │
          └─────────────────────────────────┘
                        │
           ┌────────────┼────────────┐
           ▼            ▼            ▼
      ┌────────┐   ┌────────┐   ┌────────┐
      │Node 01 │   │Node 02 │   │Node 03 │
      │実行中  │   │実行中  │   │空き    │
      └────────┘   └────────┘   └────────┘
```

**代表的なジョブスケジューラ**:

| 名前 | 特徴 | 採用例 |
|------|------|--------|
| **Slurm** | オープンソース、最も普及 | 富岳、多くの大学・研究機関 |
| **PBS Pro** | 商用版あり、安定性重視 | NASA、気象機関 |
| **LSF** | IBM製、大規模向け | 金融機関、企業 |

### 🆚 Open OnDemand と JupyterHub の比較

| 観点 | Open OnDemand | JupyterHub |
|------|---------------|------------|
| **主目的** | HPC全般のWebアクセス | Jupyter Notebook提供 |
| **提供機能** | ファイル管理、ジョブ投入、シェル、デスクトップ、Jupyter等 | Jupyter Notebook/Lab |
| **対象ユーザー** | HPC利用者全般 | データサイエンティスト |
| **導入複雑度** | 中程度 | 比較的簡単 |
| **カスタマイズ性** | 高い（独自アプリ追加可能） | 中程度 |
| **相互運用** | JupyterをアプリとしてホスティングA可能 | 単体で動作 |

## 📖 用語集（高校生向け解説付き）

| 用語 | 読み方 | 説明 |
|------|--------|------|
| **HPC** | エイチピーシー | High Performance Computing。スーパーコンピュータを使った超高速計算のこと。スマホゲームを1台のスマホで動かすのに対し、HPCは何千台ものコンピュータが協力して巨大な問題を解く。 |
| **クラスタ** | くらすた | 複数のコンピュータをネットワークで繋いで一つの大きなコンピュータのように使う仕組み。「ぶどうの房」のように多くのマシンがまとまっている。 |
| **ノード** | のーど | クラスタを構成する個々のコンピュータ。「ぶどうの粒」一つ一つに相当。 |
| **ジョブ** | じょぶ | HPCに投入する計算タスク。「計算のお仕事」のこと。レストランの注文伝票のようなもの。 |
| **ジョブスケジューラ** | じょぶすけじゅーら | 計算資源を効率よく割り当てるプログラム。レストランで注文を受けて厨房に指示を出すマネージャーのような役割。 |
| **Slurm** | すらーむ | 最も広く使われているオープンソースのジョブスケジューラ。「Simple Linux Utility for Resource Management」の略。 |
| **Open OnDemand** | おーぷん おんでまんど | WebブラウザでHPCを使えるようにするオープンソースソフトウェア。動画配信サービスのようにHPCを「オンデマンド」で使える。 |
| **Webポータル** | うぇぶぽーたる | 様々なサービスへの「入口」となるWebサイト。Yahoo!やGoogleのホームページのように、一つの場所から色々なサービスにアクセスできる。 |
| **ダッシュボード** | だっしゅぼーど | 重要な情報をまとめて表示する画面。車のダッシュボード（速度計、燃料計など）のように、状況を一目で把握できる。 |
| **インタラクティブアプリ** | いんたらくてぃぶあぷり | ユーザーが操作しながらリアルタイムに結果を見られるアプリケーション。ゲームのように「操作→即反応」が起きる。 |
| **バッチジョブ** | ばっちじょぶ | 一度投入したら完了まで自動で実行されるジョブ。洗濯機にスタートボタンを押すと終わるまで放っておけるのと同じ。 |
| **Ruby on Rails** | るびー おん れいるず | Webアプリケーションを作るためのフレームワーク。料理でいう「レシピ本」のようなもので、決まったパターンに従えば効率よくアプリが作れる。 |
| **Passenger** | ぱっせんじゃー | Webアプリを動かすためのサーバーソフト。空港の乗客を目的地に運ぶ旅客機のように、リクエストをアプリに届ける。 |
| **NGINX** | エンジンエックス | 高速なWebサーバー。多くのお客さんを効率よくさばくレストランの受付係のような存在。 |
| **Apache** | あぱっち | もう一つの有名なWebサーバー。NGINXより歴史が長く、様々な機能を持つ。 |
| **認証** | にんしょう | ユーザーが本人であることを確認すること。学校の学生証を見せて本人確認するようなもの。 |
| **OIDC** | オーアイディーシー | OpenID Connect。Webサービスでの認証の標準規格。「Googleでログイン」「Facebookでログイン」の仕組み。 |
| **SSH** | エスエスエイチ | Secure Shell。暗号化された安全な通信で遠くのコンピュータを操作する方法。鍵のかかった専用トンネルで通信するイメージ。 |
| **BoF** | ビーオーエフ | Birds of a Feather。同じ関心を持つ人たちの非公式な集まり。「類は友を呼ぶ」から来た言葉。 |
| **NSF** | エヌエスエフ | National Science Foundation。アメリカの科学技術研究を支援する政府機関。日本の文部科学省の研究助成部門に相当。 |
| **OSC** | オーエスシー | Ohio Supercomputer Center。オハイオ州立スーパーコンピュータセンター。Open OnDemandの開発元。 |
| **MGHPCC** | エムジーエイチピーシーシー | Massachusetts Green High Performance Computing Center。マサチューセッツ州の環境に配慮したHPCセンター。 |
| **Globus** | ぐろーばす | 研究データを安全に転送するためのサービス。クラウドストレージ間でファイルを移動させる「引っ越し業者」のようなもの。 |
| **JupyterHub** | じゅぴたーはぶ | 複数ユーザーでJupyter Notebookを使うためのサーバー。教室で全員がノートPCを使えるように準備する仕組み。 |
| **Jupyter Notebook** | じゅぴたー のーとぶっく | プログラムと実行結果、説明を一つのドキュメントにまとめられるツール。実験ノートのデジタル版。 |
| **RStudio** | あーるすたじお | 統計処理言語Rを使うための開発環境。Rを便利に使うための道具箱。 |
| **MATLAB** | まとらぼ | 数値計算や可視化のためのソフトウェア。工学・科学計算で広く使われる。 |
| **ColdFront** | こーるどふろんと | HPCリソースの割り当て（アロケーション）を管理するソフト。図書館の貸出管理システムのようなもの。 |
| **Open XDMoD** | おーぷん エックスディーモッド | HPCの利用状況を監視・分析するソフト。電気の使用量メーターのHPC版。 |
| **GPU** | ジーピーユー | Graphics Processing Unit。元々は画像処理用だが、AIや科学計算でも使われる高速計算チップ。 |
| **フレームワーク** | ふれーむわーく | ソフトウェア開発の「骨組み」。家を建てるときの木造軸組みのように、基本構造を提供してくれる。 |
| **オープンソース** | おーぷんそーす | ソースコード（設計図）が公開されていて、誰でも使ったり改良したりできるソフトウェア。料理のレシピを公開して皆で改良するようなもの。 |
